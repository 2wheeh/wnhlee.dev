## 함수형 컴포넌트는 클로저를 사용한다

클래스형 컴포넌트에서는 상태를 참조할 때 `this.state`을 사용한다.

반면 함수형 컴포넌트에서는 클로저를 사용한다.

???

primitive 한 상태의 경우 클래스형 컴포넌트에서는 항상 최신 값을 사용하지만 함수형 컴포넌트에서는 일종의 snapshot을 사용한다.

뭔소리지?

다음 예시를 보자

primitive한 상태를 사용하는 비동기적인 이벤트 리스너 콜백이 대기중인 상태에서 상태가 변경되면,

클래스형 컴포넌트에서는 항상 최신 값을 사용하지만 함수형 컴포넌트에서는 클로저로 primitive 값을 참조하기 떄문에 변경된 값이 아닌 이벤트가 발생한 시점의 상태를 사용한다.

이벤트가 발생한 시점의 상태를 사용하는게 더 예측가능하고 안전하게 느껴진다.

그러면 그냥 함수형 믿고 쓰면 되나?
